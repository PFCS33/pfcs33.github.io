# What is Git

  • 完全分布式的，基于快照存储的版本控制系统（DVCS）
    • Git think its data more like a stream of snapshots
      • snapshots 是对仓库的所有文件都进行照相
      • 从远程 clone 时，默认也是对该 project 的所有历史版本，全部文件进行完成克隆
    • You can perceive git as a mini filesystem with some incredibly powerful tools built on top of it

# 特点

  • OFFLINE：基本上所有操作都在本地进行
  • Integrity：
    • Everything in Git is checksummed before it is stored, and can be referred to by checksum
      • using SHA-1 hash
      • 所有数据错误都能被 Git 发现
  • Generally olny add data
  • 3 种状态
    • 文件
      • modifed: 文件在 working directory 被修改，但没有提交到 stage area
      • staged：文件被修改，并且被提交到 stage area
      • committed：文件已经被存储在 Git repository 中
    • Git project:
      • the working tree: s a single checkout of one version of the project
        • These files are pulled out of the compressed database in the Git directory and placed on disk for you to use or modify
      • the stage area:
        • 本质上，a file in the Git directory
        • 记录着下次 commit 时要记录到 snapshot 中的文件信息
      • the Git directory:
        • where Git stores the metadata and object database for your project
        • 一但文件 Commit，就意味着被 permanently 存储到 Git directory 中了
  • 两种工作方式
    • command line: 所有功能
    • graphic interface: 部分功能可用，简单

# Git Setup

# configuration variables

  • they control all aspects of how Git looks and operates
    • You can customize your Git environment by changing this vars
    • stored in 3 places
      • Linux
        • [path]/etc/gitconfig file
          • applied to every user on the system and all their repositories
          • 添加参数 --system
        • ~/.gitconfig or ~/.config/git/config file
          • applied to specific user
          • 添加参数 --globla
        • .git/config (in the Git directory)
          • applied to specific repository
          • by default, --local
      • windows
        • system 设置通常在 Git 安装的文件目录下
      • 注意：Each level overrides values in the previous level
  • use git config to change these vars
  • git config --list --show-origin
    • view all of your settings and where they are coming from

# Identity

  • user name & email address
    • every Git commit uses this information
    • 推荐 --global 参数
    • user.name , user.email
git config --global user.name aaa
git config --global user.email aaa@gmail.com

# editor

  • core.editor
  • 设置 vscode 作为默认文本编辑器 git config --global core.editor "code --wait"

# default branch name

  • By default Git will create a branch called master when you create a new repository with git init
  • init.defaultBranch
  • git config --global init.defaultBranch main

# Checking Your Settings

  • git config --list
  • You may see keys more than once, because Git reads the same key from different files
    • In this case, Git uses the last value for each unique key it sees
  • git config <key>
    • check what Git thinks a specific key’s value
  • --show-origin
    • query Git as to the origin for that value

# Git help

  • get the comprehensive manual page:
git help <verb>
git <verb> --help
man git-<verb>
  • just need a quick refresher on the available options: -h
    • E. git add -h
